INSTALLATION & RELEASE NOTES

Installation and Use

To install Interfascia, place the 'interfascia' folder in the libraries folder in your Processing directory. This will allow Processing to include Interfascia in the "Import Library" menu. (If it doesn't show up, you may need to restart Processing.)

To use Interfascia in a Processing sketch, select it in the "Import Library" menu. This will add the line import interfascia.*; to the top of your sketch file.

 

Bug Reporting

If (when) you encounter a bug, please resist the urge to email the developers directly. First, search the bug database for your issue. Chances are, it's already been discovered and some nice person has posted a temporary work-around. If it hasn't been documented, enter it as a new bug in the database. This makes it easier for us to keep track of bugs that need to be fixed for the next release. We will also try to give you feedback about workarounds or the status of the bug.

Once again, the bug database is at http://code.google.com/p/interfascia/issues/list

 

Release Notes

Interfascia 003 -- 30 August 2006

  • remove() methods are now implemented in GUIController and IFRadioController
  • Fixed a possible issue with keyboard focus getting out of sync
  • Shift + Tab reverses keyboard focus traversal
  • IFTextField now supports content strings that are longer than the width of its bounding box

Interfascia 002 -- 26 March 2006

  • Now compatible with Processing BETA
  • Draw methods play nicely with graphics in PApplet
  • Action listeners can be any Object that has an actionPerformed() method.
  • Improved font legibility
  • Tab key now traverses GUI components
  • Added IFLabel, IFTextField
